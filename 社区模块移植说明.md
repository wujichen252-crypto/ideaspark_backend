# 社区模块移植说明

## 移植内容概述

将同事项目 (`ideasparkbackend`) 的社区功能模块移植到基础项目 (`ideaspark_backend`) 中。

## 移植的组件

### 1. 新建 Repository 接口

| 文件路径 | 说明 |
|---------|------|
| `repository/CommunityCommentRepository.java` | 评论数据访问层 |
| `repository/CommunityPostLikeRepository.java` | 帖子点赞数据访问层 |
| `repository/CommunityCommentLikeRepository.java` | 评论点赞数据访问层 |
| `repository/UserFollowRepository.java` | 用户关注数据访问层 |
| `repository/CommunityGroupRepository.java` | 社区圈子数据访问层 |
| `repository/CommunityGroupMemberRepository.java` | 圈子成员数据访问层 |

| 文件路径 | 说明 | 基础路径 |
|---------|------|---------|
| `controller/CommunityPostController.java` | 帖子管理 | `/api/community/posts` |

| `controller/CommunityLikeController.java` | 点赞功能 | `/api/community/likes` |
| `controller/UserFollowController.java` | 关注功能 | `/api/follows` |

| 文件路径 | 说明 |
|---------|------|
| `util/ResponseUtil.java` | 添加 `success()`, `created()`, `error()` 方法 |

## 适配说明

### 实体类差异处理

同事项目使用简单的ID字段存储关联关系，基础项目使用JPA关联（`@ManyToOne`）。移植时进行了以下适配：

1. **外键关联**: 将同事项目中的 `authorId`, `postId` 等字段改为使用 `User`, `CommunityPost` 等实体对象
2. **ID类型**: 统一使用基础项目的ID类型（`String` 类型的UUID）
3. **用户认证**: 从 `HttpServletRequest` 中获取当前登录用户ID（`request.getAttribute("userId")`）

### 权限检查

所有需要登录的接口都添加了用户认证检查：
- 从 `HttpServletRequest` 中获取 `userId` 属性
- 未登录返回 401 错误
- 操作他人资源时返回 403 错误

## 注意事项

1. **用户认证**: 确保 `JwtAuthenticationInterceptor` 正确设置了 `userId` 属性到 request 中
2. **数据库表**: 基础项目已包含所有必要的实体类和数据库表
3. **接口路径**: 部分接口路径与同事项目不同，以符合基础项目的命名规范
