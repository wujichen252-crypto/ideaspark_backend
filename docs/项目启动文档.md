## IdeaSpark 项目启动指南
### 环境要求
- Java版本 : JDK 17+
- 数据库 : MySQL 5.7+
- 操作系统 : Windows/Linux/macOS
### 1. 开发环境启动方式 方式一：使用 Maven 直接启动（推荐）
```
# 进入项目目录
cd d:\大学就业指导\项目相关\项目落地（全栈）
\传智杯项目
\backend\backend\ideaspark_backend

# 编译并启动项目
mvn spring-boot:run
``` 方式二：先编译后启动
```
# 编译项目
mvn clean compile

# 运行项目
mvn exec:java -Dexec.mainClass="com.
ideaspark.project.IdeaSparkApplication"
``` 方式三：打包成JAR后启动
```
# 打包项目
mvn clean package -DskipTests

# 运行JAR包
java -jar target/ideaspark-0.0.
1-SNAPSHOT.jar
```
### 2. 生产环境启动方式 使用Docker启动（推荐）
```
# 构建Docker镜像
docker build -t ideaspark-backend .

# 运行容器
docker run -d -p 8081:8081 \
  -e DB_URL=jdbc:mysql://
  your-mysql-host:3306/ideaspark \
  -e DB_USERNAME=your-username \
  -e DB_PASSWORD=your-password \
  -e JWT_SECRET=your-jwt-secret \
  ideaspark-backend
``` 使用systemd服务（Linux）
```
# 创建服务文件
sudo nano /etc/systemd/system/
ideaspark.service

# 内容如下：
[Unit]
Description=IdeaSpark Backend Service
After=network.target

[Service]
Type=simple
User=ideaspark
WorkingDirectory=/opt/ideaspark
ExecStart=/usr/bin/java -jar 
ideaspark-0.0.1-SNAPSHOT.jar
Restart=always

[Install]
WantedBy=multi-user.target

# 启动服务
sudo systemctl daemon-reload
sudo systemctl enable ideaspark
sudo systemctl start ideaspark
```
### 3. 环境变量配置
项目使用 .env 文件管理配置，请创建 .env 文件：

```
# 在项目根目录创建 .env 文件
touch .env
```
.env 文件内容示例：

```
# 数据库配置
DB_URL=jdbc:mysql://localhost:3306/
ideaspark?useSSL=false&
serverTimezone=Asia/Shanghai
DB_USERNAME=root
DB_PASSWORD=your-password

# JWT配置
JWT_SECRET=your-32-character-jwt-secret
-key-here

# OSS配置（可选，如不使用文件上传功能可不配
置）
OSS_ACCESS_KEY_ID=your-access-key
OSS_ACCESS_KEY_SECRET=your-secret-key
```
### 4. 验证启动成功
启动成功后，可以通过以下方式验证：

```
# 健康检查
curl http://localhost:8081/ping
# 预期返回：{"status":200,
"message":"pong"}

# API文档
# 访问：http://localhost:8081/
swagger-ui.html
# 或：http://localhost:8081/v3/api-docs
```
### 5. 常用开发命令
```
# 清理并重新编译
mvn clean compile

# 运行测试
mvn test

# 跳过测试打包
mvn clean package -DskipTests

# 查看依赖树
mvn dependency:tree

# 检查代码质量
mvn checkstyle:check
```
### 6. 端口配置
默认端口为 8081 ，如需修改可在 application.yml 中配置：

```
server:
  port: 8081  # 修改为您需要的端口
```
### 7. 数据库初始化
项目使用Flyway进行数据库迁移，启动时会自动执行SQL脚本：

```
# 手动执行数据库迁移（如果需要）
mvn flyway:migrate
```
### 8. 日志查看
```
# 查看实时日志（如果使用systemd）
sudo journalctl -u ideaspark -f

# 查看日志文件（如果使用文件日志）
tail -f logs/ideaspark.log
```
### 📋 启动检查清单
- 确认Java版本为17+
- 确认MySQL服务已启动
- 创建并配置 .env 文件
- 确保端口8081未被占用
- 数据库连接信息正确
- JWT密钥已设置（至少32位）
### 🚨 常见问题解决
问题1 : 端口被占用

```
# 查找占用端口的进程
netstat -ano | findstr :8081
# 杀死进程（替换PID）
taskkill /PID 1234 /F
```
问题2 : 数据库连接失败

- 检查MySQL服务是否启动
- 确认数据库连接信息正确
- 检查防火墙设置
问题3 : JWT密钥错误

- 确保JWT密钥长度至少32位
- 检查 .env 文件格式是否正确
项目启动成功后，您可以通过访问 http://localhost:8081 查看主页，或访问 http://localhost:8081/swagger-ui.html 查看API文档。
