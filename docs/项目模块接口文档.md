# 项目模块接口文档

## 📋 文档说明

- **Base URL**: `/api/projects`
- **认证方式**: Bearer Token (JWT)
- **响应格式**: `ResponseEntity<Map<String, Object>>`
- **统一成功响应格式**：

json

```
{
  "code": 200,
  "message": "success",
  "data": { /* 业务数据 */ },
  "timestamp": 1621234567890
}
```

## 📊 接口汇总（共8个接口）

### 1. 项目管理接口（5个）

- 1.1 创建项目

## 🔐 权限模型概览

### 项目角色定义：

- **负责人（owner）**：由创建项目的人自动成为负责人，拥有全部权限
- **成员（member）**：被拉入项目的团队用户，拥有部分权限

### 项目创建者权限来源（必须在团队中）：

- **团队所有者（team owner）**
- **团队管理员（team admin）**

### 权限矩阵：

| 操作               | 负责人（owner） | 成员（member） | 说明                                     |
| :----------------- | :-------------- | :------------- | :--------------------------------------- |
| 创建项目           | ✅               | ❌              | 仅团队owner/admin可创建                  |
| 编辑项目信息       | ✅               | ❌              | 修改项目名称、描述、封面、分类等基础信息 |
| 编辑项目文件       | ✅               | ✅              | 修改项目模块数据（JSON格式）             |
| 上传文件           | ✅               | ✅              | 上传项目相关资源文件                     |
| 下载文件           | ✅               | ✅              | 下载项目相关资源文件                     |
| 查询项目列表       | ✅               | ✅              | 查询自己参与的所有项目                   |
| 查询项目详情       | ✅               | ✅              | 查看项目完整信息                         |
| 删除项目           | ✅               | ❌              | 彻底删除项目                             |
| 拉团队成员加入项目 | ✅               | ❌              | 从所属团队中选择成员加入项目             |
| 移除项目成员       | ✅               | ❌              | 将成员从项目中移除（不能移除自己）       |
| 查看项目成员列表   | ✅               | ✅              | 查看所有项目成员信息                     |

## 📋 详细接口文档

## 1. 项目管理接口

### 1.1 创建项目

- **URL**: `POST /`
- **描述**: 创建新项目（仅团队owner/admin可创建），创建时只能设置name、visibility
- **权限**: 团队所有者或管理员

**请求体**：

json

```
{
  "name": "AI电商平台设计",
  "visibility": "private",
  "teamId": "team_123"
}
```



**字段说明**：

- `teamId`：必须为当前用户所在的团队ID
- `initialMembers`：可选，可同时拉团队成员加入项目
- `visibility`：可选，默认为"private"
- `allowFork`：可选，默认为false

**成功响应** (code: 201)：

json

```
{
  "code": 200,
  "message": "项目创建成功",
  "data": {
    "id": "project_abc123",
    "name": "AI电商平台设计",
    "description": "基于AI的个性化电商平台",
    "ownerId": 123,
    "teamId": "team_123",
    "category": "SaaS 产品",
    "coverUrl": "https://example.com/cover.jpg",
    "status": "active",
    "progress": 0,
    "visibility": "private",
    "allowFork": true,
    "label":"AI",
    "createdAt": "2024-01-01T00:00:00Z",
    "currentUserRole": "owner"
  },
  "timestamp": 1621234567890
}
```

### 2.2 添加项目成员（从团队中拉人）

- **URL**: `POST /{projectId}/members`
- **描述**: 从所属团队中拉人加入项目（仅负责人可操作）
- **权限**: 项目负责人
- **限制**: 只能添加当前项目所属团队中的成员

**路径参数**：

| 参数名    | 类型   | 必填 | 描述   |
| :-------- | :----- | :--- | :----- |
| projectId | String | 是   | 项目ID |

**请求体**：

json

```
{
  "userIds": [456, 789],
  "role": "member"
}
```



**字段说明**：

- `userIds`：团队成员的用户ID数组
- `role`：只能为"member"（项目角色）

**成功响应** (code: 200)：

json

```
{
  "code": 200,
  "message": "成员添加成功",
  "data": {
    "totalInvited": 2,
    "successCount": 2,
    "addedMembers": [
      {
        "userId": 456,
        "userName": "李四",
        "role": "member"
      },
      {
        "userId": 789,
        "userName": "王五",
        "role": "member"
      }
    ]
  }
}
```

### 2.3 移除项目成员

- **URL**: `DELETE /{projectId}/members/{memberId}`
- **描述**: 将成员从项目中移除（仅负责人可操作）
- **权限**: 项目负责人

**路径参数**：

| 参数名    | 类型   | 必填 | 描述       |
| :-------- | :----- | :--- | :--------- |
| projectId | String | 是   | 项目ID     |
| memberId  | String | 是   | 项目成员ID |

**移除规则**：

- 只能移除项目成员（role="member"）
- 不能移除项目负责人（role="owner"）
- 不能移除自己

**成功响应** (code: 200)：

json

```
{
  "code": 200,
  "message": "成员移除成功",
  "data": {
    "memberId": "pm_456",
    "userId": 456,
    "userName": "李四",
    "projectId": "project_abc123",
    "projectName": "AI电商平台设计"
  }
}
```

## ⚠️ 注意事项

1. **权限验证**：所有接口都需要验证用户权限，包括团队权限和项目权限
2. **数据一致性**：项目必须属于某个团队，团队成员才能被添加到项目中
3. **事务处理**：成员添加、移除等操作需要事务保证数据一致性
4. **文件操作接口**：文件上传、下载、编辑等接口需单独实现，本文档仅定义项目管理核心接口
5. **项目模块数据**：`project_modules`表的操作应单独提供模块管理接口
6. **错误处理**：
   - 403：权限不足
   - 404：项目不存在或用户不是项目成员
   - 409：重复添加团队成员
   - 400：参数验证失败
