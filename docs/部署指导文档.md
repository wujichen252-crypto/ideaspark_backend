# IdeaSpark 后端项目部署指导文档

## 一、项目概述

### 1.1 技术栈
- **框架**: Spring Boot 3.4.1
- **Java 版本**: JDK 17+
- **数据库**: MySQL 8.x
- **构建工具**: Maven
- **数据库迁移**: Flyway
- **文档**: SpringDoc OpenAPI

### 1.2 项目结构
```
ideaspark_backend/
├── src/
│   ├── main/
│   │   ├── java/com/ideaspark/       # 业务代码
│   │   └── resources/
│   │       ├── application.yml       # 主配置文件
│   │       └── db/migration/         # Flyway 迁移脚本
│   └── test/                         # 测试代码
├── pom.xml                           # Maven 配置
├── Dockerfile                        # Docker 构建文件
└── docs/                             # 文档目录
```

## 二、环境准备

### 2.1 服务器要求
- **操作系统**: Linux (CentOS 7+/Ubuntu 18.04+) 或 Windows Server
- **内存**: 最低 2GB，推荐 4GB+
- **磁盘**: 最低 20GB 可用空间
- **网络**: 开放 8080 端口（或自定义端口）

### 2.2 必需软件
| 软件 | 版本要求 | 用途 |
|------|----------|------|
| JDK | 17+ | Java 运行环境 |
| MySQL | 8.x | 数据存储 |
| Maven | 3.8+ | 项目构建（可选，也可用 wrapper） |
| Nginx | 任意 | 反向代理（生产环境推荐） |

### 2.3 安装 JDK 17

#### CentOS/RHEL
```bash
# 使用 OpenJDK
sudo yum install -y java-17-openjdk java-17-openjdk-devel

# 验证安装
java -version
```

#### Ubuntu/Debian
```bash
# 使用 OpenJDK
sudo apt update
sudo apt install -y openjdk-17-jdk

# 验证安装
java -version
```

#### Windows
1. 下载 JDK 17: https://adoptium.net/
2. 安装并配置环境变量 `JAVA_HOME`
3. 添加 `%JAVA_HOME%\bin` 到 `PATH`

## 三、数据库准备

### 3.1 创建数据库
```sql
-- 登录 MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE ideaspark 
    DEFAULT CHARACTER SET utf8mb4 
    DEFAULT COLLATE utf8mb4_unicode_ci;

-- 创建用户（推荐）
CREATE USER 'ideaspark'@'%' IDENTIFIED BY 'your_secure_password';
GRANT ALL PRIVILEGES ON ideaspark.* TO 'ideaspark'@'%';
FLUSH PRIVILEGES;
```

### 3.2 配置远程访问（如需）
```bash

# 重启 MySQL
sudo systemctl restart mysql
```

## 四、应用配置

### 4.1 环境变量说明

项目通过环境变量读取配置，**必须设置以下变量**：

| 环境变量 | 说明 | 示例 |
|----------|------|------|
| `DB_URL` | 数据库连接 URL | `jdbc:mysql://localhost:3306/ideaspark?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai` |
| `DB_USERNAME` | 数据库用户名 | `ideaspark` |
| `DB_PASSWORD` | 数据库密码 | `your_password` |
| `JWT_SECRET` | JWT 密钥（长度建议 32 位以上） | `your_random_secret_key_here_32chars` |
| `OSS_ACCESS_KEY_ID` | 阿里云 OSS AccessKey（可选） | `LTAI...` |
| `OSS_ACCESS_KEY_SECRET` | 阿里云 OSS Secret（可选） | `your_oss_secret` |

### 4.2 配置文件说明

`application.yml` 关键配置项：

```yaml
server:
  port: 8080  # 服务端口

spring:
  flyway:
    enabled: false  # 默认关闭，如需自动迁移改为 true
  datasource:
    url: ${DB_URL}
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}
    hikari:
      maximum-pool-size: 20  # 连接池最大连接数
      minimum-idle: 5        # 最小空闲连接数

jwt:
  secret: ${JWT_SECRET}
  expire-seconds: 3600  # Token 过期时间（秒）

oss:
  enabled: true  # 是否启用阿里云 OSS
```

## 五、部署方式

### 方式一：传统部署（推荐生产环境）

#### 步骤 1：构建项目
```bash
# 进入项目目录
cd ideaspark_backend

# 使用 Maven 打包（跳过测试）
mvn clean package -DskipTests

# 生成的 JAR 文件位于
# target/ideaspark-0.0.1-SNAPSHOT.jar
```

#### 步骤 2：上传服务器
```bash
# 创建应用目录
sudo mkdir -p /opt/ideaspark
sudo chown -R $USER:$USER /opt/ideaspark

# 上传 JAR 文件（使用 scp 或其他方式）
scp target/ideaspark-0.0.1-SNAPSHOT.jar user@server:/opt/ideaspark/
```

#### 步骤 3：创建启动脚本

**Linux 系统服务（systemd）**

创建服务文件：
```bash
sudo tee /etc/systemd/system/ideaspark.service << 'EOF'
[Unit]
Description=IdeaSpark Backend Service
After=network.target mysql.service

[Service]
Type=simple
User=www
WorkingDirectory=/opt/ideaspark

# 环境变量
Environment="DB_URL=jdbc:mysql://localhost:3306/ideaspark?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai"
Environment="DB_USERNAME=ideaspark"
Environment="DB_PASSWORD=your_password"
Environment="JWT_SECRET=your_jwt_secret_key_at_least_32_chars"
Environment="OSS_ACCESS_KEY_ID=your_oss_key"
Environment="OSS_ACCESS_KEY_SECRET=your_oss_secret"

# JVM 参数（根据服务器内存调整）
Environment="JAVA_OPTS=-Xms512m -Xmx1024m -XX:+UseG1GC"

ExecStart=/usr/bin/java $JAVA_OPTS -jar ideaspark-0.0.1-SNAPSHOT.jar
ExecStop=/bin/kill -15 $MAINPID

Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF
```

启动服务：
```bash
# 重载 systemd
sudo systemctl daemon-reload

# 设置开机自启
sudo systemctl enable ideaspark

# 启动服务
sudo systemctl start ideaspark

# 查看状态
sudo systemctl status ideaspark

# 查看日志
sudo journalctl -u ideaspark -f
```

**Windows 服务**

使用 NSSM 创建服务：
```powershell
# 下载 NSSM: https://nssm.cc/download
# 创建服务
nssm install IdeaSparkBackend

# 配置参数：
# Path: C:\Program Files\Java\jdk-17\bin\java.exe
# Startup directory: D:\ideaspark
# Arguments: -jar ideaspark-0.0.1-SNAPSHOT.jar

# 在 Environment 选项卡添加环境变量
# DB_URL=jdbc:mysql://localhost:3306/ideaspark?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
# DB_USERNAME=ideaspark
# DB_PASSWORD=your_password
# JWT_SECRET=your_jwt_secret

# 启动服务
nssm start IdeaSparkBackend
```

### 方式二：Docker 部署

#### 步骤 1：创建 Dockerfile
```dockerfile
FROM eclipse-temurin:17-jdk-alpine

# 设置工作目录
WORKDIR /app

# 复制 JAR 文件
COPY target/ideaspark-0.0.1-SNAPSHOT.jar app.jar

# 暴露端口
EXPOSE 8080

# 启动命令
ENTRYPOINT ["java", "-jar", "app.jar"]
```

#### 步骤 2：构建镜像
```bash
# 构建镜像
docker build -t ideaspark-backend:latest .

# 查看镜像
docker images | grep ideaspark
```

#### 步骤 3：运行容器
```bash
# 运行容器
docker run -d \
  --name ideaspark-backend \
  -p 8080:8080 \
  -e DB_URL="jdbc:mysql://host.docker.internal:3306/ideaspark?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai" \
  -e DB_USERNAME="ideaspark" \
  -e DB_PASSWORD="your_password" \
  -e JWT_SECRET="your_jwt_secret" \
  -e OSS_ACCESS_KEY_ID="your_key" \
  -e OSS_ACCESS_KEY_SECRET="your_secret" \
  --restart unless-stopped \
  ideaspark-backend:latest

# 查看日志
docker logs -f ideaspark-backend
```

#### 使用 Docker Compose（推荐）

创建 `docker-compose.yml`：
```yaml
version: '3.8'

services:
  app:
    build: .
    container_name: ideaspark-backend
    ports:
      - "8080:8080"
    environment:
      - DB_URL=jdbc:mysql://mysql:3306/ideaspark?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
      - DB_USERNAME=ideaspark
      - DB_PASSWORD=${DB_PASSWORD}
      - JWT_SECRET=${JWT_SECRET}
      - OSS_ACCESS_KEY_ID=${OSS_ACCESS_KEY_ID}
      - OSS_ACCESS_KEY_SECRET=${OSS_ACCESS_KEY_SECRET}
    depends_on:
      - mysql
    restart: unless-stopped
    networks:
      - ideaspark-network

  mysql:
    image: mysql:8.0
    container_name: ideaspark-mysql
    environment:
      - MYSQL_ROOT_PASSWORD=${MYSQL_ROOT_PASSWORD}
      - MYSQL_DATABASE=ideaspark
      - MYSQL_USER=ideaspark
      - MYSQL_PASSWORD=${DB_PASSWORD}
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql
    restart: unless-stopped
    networks:
      - ideaspark-network

volumes:
  mysql-data:

networks:
  ideaspark-network:
    driver: bridge
```

创建 `.env` 文件：
```env
MYSQL_ROOT_PASSWORD=root_password
DB_PASSWORD=ideaspark_password
JWT_SECRET=your_jwt_secret_key_at_least_32_chars
OSS_ACCESS_KEY_ID=your_oss_key
OSS_ACCESS_KEY_SECRET=your_oss_secret
```

启动：
```bash
docker-compose up -d
```

## 六、Nginx 反向代理配置

### 6.1 基础配置
```nginx
server {
    listen 80;
    server_name api.yourdomain.com;

    # 反向代理到后端
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
}
```

### 6.2 HTTPS 配置（推荐）
```nginx
server {
    listen 80;
    server_name api.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.yourdomain.com;

    # SSL 证书配置
    ssl_certificate /path/to/your/certificate.crt;
    ssl_certificate_key /path/to/your/private.key;
    ssl_session_timeout 1d;
    ssl_session_cache shared:SSL:50m;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 反向代理
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 九、常见问题排查

### 9.1 启动失败
| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 端口被占用 | 8080 端口已被使用 | 修改 `server.port` 或停止占用进程 |
| 数据库连接失败 | 环境变量配置错误或网络不通 | 检查 DB_URL、用户名密码，验证网络连通性 |
| JWT_SECRET 未设置 | 缺少必需环境变量 | 设置 JWT_SECRET 环境变量 |
| 内存不足 | JVM 堆内存设置过大 | 调整 `-Xmx` 参数 |

### 9.2 数据库问题
```bash
# 测试数据库连接
mysql -h <host> -u <user> -p -e "SELECT 1"

# 检查网络连通性
telnet <mysql-host> 3306
```

### 9.3 性能调优
```bash
# JVM 参数建议（根据服务器内存调整）
# 2GB 内存服务器
-Xms512m -Xmx1024m

# 4GB 内存服务器
-Xms1024m -Xmx2048m

# 8GB 内存服务器
-Xms2048m -Xmx4096m
```

## 十一、安全建议

1. **数据库安全**
   - 使用强密码
   - 限制数据库访问 IP
   - 定期备份数据

2. **应用安全**
   - 设置复杂的 JWT_SECRET
   - 定期更换密钥
   - 启用 HTTPS

3. **服务器安全**
   - 配置防火墙，只开放必要端口
   - 定期更新系统补丁
   - 使用非 root 用户运行应用

4. **密钥管理**
   - 不要将密钥硬编码在代码中
   - 使用环境变量或密钥管理服务
   - 定期轮换阿里云 OSS 密钥
